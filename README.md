# Dotfiles

Personal dotfiles for Vim, tmux, and zsh configuration.

## Features

### Vim
- **Plugin Manager**: Vundle
- **Color Scheme**: Solarized Dark
- **Key Plugins**:
  - NERDTree - File explorer
  - CtrlP - Fuzzy file finder
  - vim-fugitive - Git integration
  - vim-airline - Status line
  - vim-buffergator - Buffer explorer
  - ALE - Linting engine
  - UltiSnips - Snippet engine
  - vim-surround - Surround text objects
  - emmet-vim - HTML/CSS abbreviation
  - And more...

### tmux
- 256 color support
- Vim-like keybindings
- Mouse support
- Solarized dark theme
- WSL/Linux compatible (escape sequence fixes)
- UTF-8 multibyte character support

### Zsh
- **oh-my-zsh** - Framework
- **powerlevel10k** - Theme
- **Plugins**:
  - zsh-autosuggestions
  - zsh-syntax-highlighting
  - git, docker, composer, npm

### Laravel Helper Functions
- `laravel-dev` - Switch to development mode
- `laravel-prod` - Switch to production mode
- `laravel-optimize` - Clear and optimize caches
- `laravel-deploy` - Run deployment script
- `laravel-restart` - Restart PHP-FPM and Nginx
- `laravel-log` - Tail Laravel log
- `laravel-schedule-test` - List scheduled tasks

## Requirements

- Vim 7.4.849+ or Vim 8.0+ (for UltiSnips compatibility)
- git
- curl
- zsh

## Installation

```bash
# Clone the repository
git clone https://github.com/comeonly/dotfiles ~/dotfiles

# Run installer
cd ~/dotfiles
./install.sh
```

The installer will:
1. Check Vim version compatibility
2. Install Vundle and Vim plugins
3. Install oh-my-zsh and powerlevel10k
4. Install zsh plugins
5. Create symlinks for config files
6. Change default shell to zsh
7. Configure custom settings with dynamic username replacement

### Post-Installation

After installation:
1. Log out and log back in (for shell change)
2. Configure powerlevel10k: `p10k configure`
3. Install [Nerd Fonts](https://github.com/ryanoasis/nerd-fonts) for powerline symbols

## Update

```bash
cd ~/dotfiles
./update.sh
```

Updates:
- dotfiles repository
- oh-my-zsh and plugins
- powerlevel10k theme
- Vundle and Vim plugins

## Uninstall

```bash
cd ~/dotfiles
./uninstall.sh
```

The uninstaller will:
- Remove all symlinks
- Restore backup files (if available)
- Optionally remove oh-my-zsh
- Optionally remove ~/.vim directory
- Optionally change shell back to bash

## File Structure

```
dotfiles/
├── .vimrc                  # Vim configuration
├── .vimrc.before          # Pre-configuration (optional)
├── .tmux.conf             # tmux configuration
├── .zshrc.custom          # Custom zsh configuration (template)
├── install.sh             # Installation script
├── update.sh              # Update script
├── uninstall.sh           # Uninstall script
└── README.md              # This file
```

## Configuration Files

### Vim
- `~/.vimrc` - Main configuration (symlink)
- `~/.vim/bundle/` - Plugins installed by Vundle
- `~/.vim/_undodir/` - Undo history

### tmux
- `~/.tmux.conf` - Configuration (symlink)

### Zsh
- `~/.zshrc` - Generated by oh-my-zsh
- `~/.zshrc.custom` - Custom configuration (username replaced)
- `~/.p10k.zsh` - powerlevel10k configuration

## Customization

### Adding Vim Plugins

Edit `~/.vimrc` and add:
```vim
Plugin 'username/plugin-name'
```

Then run:
```bash
vim +PluginInstall +qall
```

### Modifying Laravel Functions

Edit `.zshrc.custom` in the dotfiles directory, then run:
```bash
./update.sh
```
